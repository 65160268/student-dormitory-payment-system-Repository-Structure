# 📑 Software Requirements Specification (SRS)

## Project: Student Dormitory Payment and Billing Management System

**Team: JR** | **Status: MVP พร้อมสาธิตและอยู่ระหว่างปรับเอกสารขั้นสุดท้าย**

---

## 1. บทนำ (Introduction)

### 1.1 วัตถุประสงค์ (Purpose)

เพื่อพัฒนาระบบสารสนเทศสำหรับบริหารจัดการการจัดเก็บค่าเช่าหอพักนักศึกษา ช่วยลดขั้นตอนการทำงานด้วยมือ (Manual Process) และเพิ่มความสะดวก รวดเร็ว โปร่งใส ให้กับทั้งนักศึกษาและเจ้าหน้าที่ที่เกี่ยวข้อง

### 1.2 ขอบเขตของระบบ (Scope)

ระบบรองรับวงจรการบริหารจัดการหอพักตั้งแต่ต้นจนจบ ดังนี้:

* การจัดการบัญชีผู้ใช้และข้อมูลการเข้าพัก
* การบันทึกมิเตอร์น้ำ-ไฟรายเดือน
* การแจ้งชำระเงินผ่านระบบออนไลน์และการอัปโหลดหลักฐาน (Slip)
* การตรวจสอบและอนุมัติยอดเงินโดยเจ้าหน้าที่การเงิน
* การจัดการรายการ overdue, export รายงานการเงิน และ audit log สำหรับงานสำคัญ
* การแจ้งซ่อม/ติดตาม maintenance request ภายในระบบ
* การสรุปรายงานรายรับและสถิติต่างๆ สำหรับผู้บริหาร

### 1.3 ผู้ใช้งานระบบ (Target Users)

ระบบนี้เชื่อมโยงผู้ใช้งาน 5 กลุ่มหลักเข้าด้วยกัน:

1. **นักศึกษา:** ผู้เช่าพักและผู้ชำระเงิน
2. **บุคลากร (เจ้าหน้าที่หอพัก):** ผู้ดูแลความเรียบร้อยและบันทึกข้อมูลมิเตอร์
3. **เจ้าหน้าที่การเงิน:** ผู้ตรวจสอบยอดเงินและอนุมัติใบเสร็จ
4. **ผู้จัดการ:** ผู้กำกับดูแลภาพรวม
5. **ฝ่ายบริหาร:** ผู้เรียกดูรายงานเพื่อการตัดสินใจ

## 2. คำอธิบายโดยรวม (Overall Description)

### 2.1 มุมมองของระบบ (System Perspective)

ระบบเป็นแพลตฟอร์มเว็บแอปพลิเคชันที่ใช้การเข้าถึงข้อมูลตามบทบาทหน้าที่ (Role-Based Access Control) เพื่อรักษาความปลอดภัยของข้อมูลส่วนบุคคลและข้อมูลทางการเงิน

### 2.2 ฟังก์ชันหลักของระบบ (System Functions)

| บทบาท (Role) | ฟังก์ชันที่รับผิดชอบ |
| --- | --- |
| **นักศึกษา** | เข้าสู่ระบบ, ตรวจสอบยอดค้างชำระ, อัปโหลดสลิปแจ้งชำระเงิน, ดูประวัติการชำระ และแจ้งซ่อม |
| **เจ้าหน้าที่หอพัก** | บันทึกเลขมิเตอร์น้ำ-ไฟ, จัดการข้อมูลย้ายเข้า-ออก, อัปเดตสถานะงานซ่อม |
| **เจ้าหน้าที่การเงิน** | ตรวจสอบ Slip, อนุมัติ/ปฏิเสธการชำระ, ออกใบแจ้งหนี้รายเดือน, ติดตาม overdue และ export รายงาน |
| **ผู้บริหาร** | ดู dashboard รายงานสรุปรายได้ประจำเดือน/ปี และสถิติการค้างชำระเงิน |
| **ผู้ดูแลระบบ (Admin)** | จัดการข้อมูลห้องพัก/ราคาเช่า, จัดการสิทธิ์และบัญชีผู้ใช้, ตรวจสอบ audit log |

### 2.3 ข้อจำกัด (Constraints)

* นักศึกษาต้องเข้าสู่ระบบผ่าน **รหัสนิสิต** ที่มีอยู่ในฐานข้อมูลเท่านั้น
* ระบบ production ปัจจุบัน deploy แบบ Docker container เดียวบน Proxmox LXC เพื่อให้การสาธิตและการดูแลระบบง่ายขึ้น
* ฟีเจอร์เชิงขยายบางส่วน เช่น external notification และ E-Receipt PDF อยู่ใน roadmap ระยะถัดไป

## 3. ความต้องการเฉพาะ (Specific Requirements)

### 3.1 ความต้องการเชิงหน้าที่ (Functional Requirements)

| รหัส (ID) | คำอธิบายความต้องการ (Requirement Description) |
| --- | --- |
| **REQ-001** | ระบบต้องรองรับการเข้าสู่ระบบด้วยรหัสนิสิต |
| **REQ-002** | ระบบต้องสามารถแสดงยอดค่าน้ำ ค่าไฟ และค่าเช่าแบบ Real-time |
| **REQ-003** | ระบบต้องมีฟังก์ชันอัปโหลดหลักฐานการโอนเงิน (Slip Verification) |
| **REQ-004** | ระบบต้องบันทึกเลขมิเตอร์น้ำและไฟรายเดือนแยกตามห้องพักได้ |
| **REQ-005** | ระบบต้องสามารถสร้างใบแจ้งหนี้รายเดือนตามข้อมูลห้องพักและมิเตอร์ได้ |
| **REQ-006** | ระบบต้องให้ฝ่ายการเงินอนุมัติหรือปฏิเสธรายการชำระเงินได้ |
| **REQ-007** | ระบบต้องแสดงสถิติการค้างชำระเงินในรูปแบบรายงานที่เข้าใจง่าย |
| **REQ-008** | ระบบต้องรองรับการจัดการบัญชีผู้ใช้และการกำหนดสิทธิ์ตามบทบาท |
| **REQ-009** | ระบบต้องรองรับการแจ้งซ่อมและอัปเดตสถานะงานซ่อมภายในระบบ |
| **REQ-010** | ระบบต้องรองรับการ export ข้อมูลรายงานการเงินและการตรวจสอบ audit log |

### 3.2 ความต้องการที่ไม่ใช่เชิงหน้าที่ (Non-functional Requirements)

* **Security:** ระบบต้องมีการจัดการสิทธิ์และบัญชีผู้ใช้งานแยกตามบทบาทอย่างเคร่งครัด ข้อมูลทางการเงินต้องได้รับการเข้ารหัส
* **Performance:** ระบบต้องประมวลผลรายงานสรุปรายได้ประจำปีได้ภายในเวลาไม่เกิน 5 วินาที

## 4. อินเทอร์เฟซภายนอก (External Interface Requirements)

### 4.1 User Interface (UI)

* **Student Portal:** ออกแบบให้ใช้งานง่ายบนมือถือ (Mobile First) เพื่อความสะดวกในการตรวจยอด, แจ้งชำระ และแจ้งซ่อม
* **Staff/Finance/Admin Portal:** เน้นรูปแบบตาราง (Data Grid) และ dashboard เพื่อค้นหา อนุมัติ และบริหารข้อมูลได้รวดเร็ว

### 4.2 Software Interface

* ระบบปัจจุบันเชื่อมต่อกับ **MySQL/MariaDB** ผ่าน Drizzle ORM และรองรับ fallback demo data เมื่อไม่ได้ตั้งค่า database
* การเชื่อมต่อกับระบบภายนอกเพิ่มเติม เช่นทะเบียนนิสิตกลางหรือ notification service ถูกกำหนดเป็นส่วนขยายในอนาคต

---

## 5. ฟีเจอร์ที่สำคัญ (Key Features)

* **Smart Finance Management:** ระบบตรวจสอบสลิป, อนุมัติ/ปฏิเสธรายการชำระ และติดตาม overdue ได้ในหน้าจอเดียว
* **Executive Insights:** รายงานเชิงวิเคราะห์ที่แสดงรายรับ, ยอดค้างชำระ และสถานะห้อง เพื่อช่วยในการตัดสินใจ
* **Operational Support:** รองรับ maintenance requests, room assignment/move-out และการจัดการ room rates / audit logs สำหรับงานปฏิบัติการ
