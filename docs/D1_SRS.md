บทนำ
	วัตถุประสงค์ : เพื่อพัฒนาระบบสารสนเทศสำหรับบริหารจัดการการจัดเก็บค่าเช่าหอพักนักศึกษา ช่วยลดขั้นตอนการทำงานด้วยมือ และเพื่มความสะดวกให้กับทั้งนักศึกษาและเจ้าหน้าที่
	ขอบเขต : ระบบรอรับการจัดการโปรไฟล์นักศึกษา การบันทึกมิเตอร์น้ำ-ไฟ การแจ้งชำระเงินผ่านระบบออนไลน์ การตรวจสอบหลักฐานการโอน และการออกรายงานสรุปสำหรับผู้บริหาร
	ภาพรวม : ระบบจะเชื่อมโยงกับผู้ใช้งาน 5 กลุ่มหลัก นักศึกษา, บุคลากร, เจ้าหน้าที่การเงิน, ผู้จัดการ และ ฝ่ายบริหาร เข้าด้วยกันเพื่อจัดการวงจรการเช่าพักตั้งแต่การย้ายเข้าจนถึงการสรุปรายได้ประจำปี

คำอธิบายโดยรวม
	มุมมองของระบบ : ระบบแบ่งระดับการเข้าถึงตามบทบาทหน้าที่ เพื่อความปลอดภัยของข้อมูล

ฟังก์ชั่นหลักของระบบ : ตามฟังก์ชั่นที่ออกแบบไว้แบ่งออกตามผู้ใช้งาน ดังนี้
นักศึกษา : จัดการโปรไฟล์, ตรวจสอบยอดค้างชำระ, แจ้งชำระเงิน(อัปโหลดสลิป), ดูประวัติและดาวน์โหลด ใบเสร็จอิเล็กทรอนิกส์
เจ้าหน้าที่หอพัก : บันทึกเลขมิเตอร์น้ำ-ไฟ, จัดการข้อมูลการย้ายเข้า-ออก และส่งการแจ้งเตือนยอดค้างชำระ
เจ้าหน้าที่การเงิน : ตรวจสอบและอนุมัติหลักฐานการโอนเงิน, ยืนยันยอดเงินเข้าบัญชี และออกใบแจ้งหนี้
ผู้บริหาร : ดูรายงานสรุปรายได้ประจำเดือน/ปี และดูสถิติการค้างชำระเงิน
ผู้ดูแลระบบ : จัดการข้อมูลห้องพัก/ราคาเช่า และจัดการสิทธิ์ผู้ใช้งาน

รหัส (ID)
คำอธิบายความต้องการ (Requirement Description)
REQ-001
ระบบต้องรองรับการเข้าสู่ระบบด้วยรหัสนิสิต
REQ-002
ระบบต้องสามารถแสดงยอดค่าน้ำ ค่าไฟ และค่าเช่าแบบ Real-time
REQ-003
ระบบต้องมีฟังก์ชันอัปโหลดหลักฐานการโอนเงิน (Slip)
REQ-004
ระบบต้องบันทึกเลขมิเตอร์น้ำและไฟรายเดือนได้
REQ-005
ระบบต้องสามารถสร้างใบเสร็จรับเงินอิเล็กทรอนิกส์ (E-Receipt)
REQ-006
ระบบต้องสามารถออกใบแจ้งหนี้ (Invoice) รายเดือนได้
REQ-007
ระบบต้องแสดงสถิติการค้างชำระเงินในรูปแบบรายงาน

	
ความต้องการที่ไม่ใช่เชิงหน้าที่
ความปลอดภัย (Security): ระบบต้องมีการจัดการสิทธิ์และบัญชีผู้ใช้งานแยกตามบทบาท
ประสิทธิภาพ (Performance): ระบบควรสามารถออกรายงานสรุปรายได้ประจำปีได้อย่างรวดเร็ว

ความต้องการอินเทอร์เฟซภายนอก
User Interface: รองรับการแสดงผลทั้งสำหรับเจ้าหน้าที่ (Back-end) และนักศึกษา (Front-end)
Software Interface: ระบบอาจต้องเชื่อมต่อกับระบบฐานข้อมูลทะเบียนนิสิต (อ้างอิงจากการเข้าสู่ระบบด้วยรหัสนิสิต)

ฟีเจอร์ระบบ
ระบบจัดการการเงิน: เจ้าหน้าที่การเงินสามารถอนุมัติ Slip และยืนยันยอดเงิน เพื่อเปลี่ยนสถานะการชำระเงินของนักศึกษาอัตโนมัติ
ระบบรายงานผู้บริหาร: แสดงแนวโน้มรายได้และปัญหาการค้างชำระ เพื่อใช้ในการวางแผนบริหารจัดการหอพัก
