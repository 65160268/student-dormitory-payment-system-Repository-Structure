พื่อลดความผิดพลาดในการคำนวณค่าน้ำ-ค่าไฟรายห้อง
เพื่อเพิ่มความสะดวกในการเข้าถึงข้อมูลการเงินของนิสิต
เพื่อสร้างระบบจัดเก็บหลักฐานการโอนเงินที่ค้นหาง่าย

เกณฑ์สำเร็จ

TOP 5 RISKS
1. RISK: Data Privacy Breach (ข้อมูลการเงินรั่วไหล)
ความรุนแรง : สูงมาก 
ความเป็นไปได้ : ต่ำ (10%) 
Score : 10% × 95 = 9.5 (Low Prob / Critical Impact) 
ผลกระทบ : ผิดกฎหมาย PDPA และเสียความเชื่อมั่นอย่างรุนแรง
การบรรเทา : เข้ารหัสข้อมูลสลิปและจำกัดสิทธิ์เจ้าหน้าที่ในการดูข้อมูลส่วนตัวนิสิต 
ผู้รับผิดชอบ : นิรัตติศัย (Dev)

2. RISK: Billing Calculation Error (คำนวณยอดเงินผิด)
ความรุนแรง : สูง
ความเป็นไปได้ : ปานกลาง (40%)
Score : 20% x 90 = 18 (Medium)
ผลกระทบ :  นิสิตจ่ายเงินไม่ตรงตามจริง เกิดการร้องเรียนเป็นจำนวนมาก
การบรรเทา: ทำ Automated Unit Test สำหรับสูตรคำนวณ (Unit x Rate) ทุกครั้งที่มีการอัปเดตโค้ด
ผู้รับผิดชอบ : อำนาจ (Dev)

3. RISK: Fake Slip Upload (การปลอมแปลงสลิป)
ความรุนแรง : ปานกลาง 
ความเป็นไปได้ : ปานกลาง (30%)
Score : 30% × 80 = 24 (High)
ผลกระทบ : มหาวิทยาลัยสูญเสียรายได้ และเสียเวลาเจ้าหน้าที่ในการตรวจสอบ
